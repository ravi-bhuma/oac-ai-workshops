Dataset Preparation
10 + 5 Q&A

The AI Assistant and AI Agents rely on indexed metadata to understand your data. Without proper indexing, the AI doesn't know which columns exist, what values they contain, or how users refer to them. Think of it as teaching the AI your data's vocabulary.

Hands-On Steps
  1. Navigate to Datasets → right-click your dataset → Inspect
  2. Click the Search tab
  3. Under "Index Dataset For", select "Assistant"
  4. For each column, choose Index Type:
    • Name & Values — for categorical columns users will filter on (Country, Product Category, City)
    • Name only — for measures and high-cardinality fields (Revenue, Profit, Order ID)
  5. Add Synonyms for key fields:
    Revenue → "sales", "income", "turnover" Customer Segment → "customer type", "buyer category" Region → "territory", "area"
  6. Click Save → Run Now
  7. Check indexing status until "Success"
Best Practices: Use descriptive column names (not "Ename" → "Employee Name"). Only index columns users will ask about. Exclude technical fields (IDs, keys). Disambiguate similar date columns. Replace null values with "Unknown". Avoid special characters in attributes.

Homepage AI Assistant
10 + 5 Q&A

The Homepage AI Assistant is the front door to analytics. Users start with intent, not dashboards — asking questions or requesting complete workbooks directly from the OAC Home page.

Key Capabilities

  • Instant workbook creation — generate complete dashboards from prompts like "Build a profitability dashboard from Sample Sales"
  • Context-aware dialogue — follow-up questions retain intent and context
  • Smart search — combines catalog discovery with semantic understanding
Live Demo Walkthrough
  1. From OAC Home page, locate the search/assistant bar
  2. Type: "Create a sales performance dashboard showing revenue by region and product category"
  3. Watch the AI generate a complete workbook with visualizations
  4. Ask follow-up: "Add a trend line for the last 4 quarters"
  5. Save the generated workbook

Workbook AI Assistant
20 + 5 Q&A

The Workbook AI Assistant lives inside the workbook authoring experience, helping users create and refine visualizations through natural language conversation.

How It Works Under the Hood
  1. User question received — OAC captures the natural language query
  2. Dataset identification — matches indexed metadata
  3. Prompt construction — builds prompt with metadata, columns, synonyms, and question
  4. LLM processing — translates question into a structured query
  5. Query verification — validates against data model and executes
  6. Visualization generation — produces the appropriate chart
Hands-On Lab (15 min)
  1. Open or create a workbook on your indexed dataset
  2. Click the AI Assistant in the top bar
  3. Ask sample questions one by one:
    • "What are the top customer segments by sales?"
    • "How does order priority affect shipping costs?"
    • "Show revenue by product category and region"
    • "Compare Q3 vs Q4 sales by territory"
  4. Interact with results: Change chart type, add/remove fields, ask iterative follow-ups
  5. Click + to add best visualizations to your workbook canvas
Enable for Consumer Users
  1. Click the Present tab in your workbook
  2. Under Insights Panel, toggle Workbook Assistant → On
  3. Consumer users can now use the Assistant when viewing this workbook

AI Agents
30 + 5 Q&A

AI Agents are domain-specific experts that combine enterprise data with custom instructions and knowledge documents using RAG. Without proper engineering, agents behave like generic LLMs — they don't know your fiscal calendar, your KPI formulas, or what "important orders" means.

The 4 Agent Components

1. Dataset

The facts. One indexed dataset per agent. Configure column subsetting and agent filters.

2. Supplemental Instructions

The brain. Up to 6,000 characters of custom rules using the R.T.C.C.O.E Framework.

3. Knowledge Documents

The reference library. Up to 10 PDFs/TXTs (5MB each), processed via RAG.

4. Welcome Message

The introduction. Sets expectations and provides sample questions for users.

R.T.C.C.O.E Framework

Structure your Supplemental Instructions using this 6-component framework:

R
Role
"You are a Senior Sales Analytics Expert"
T
Task
"Answer with charts, highlight anomalies"
C
Context
Fiscal calendars, KPI formulas, hierarchies
C
Constraints
Default metrics, mandatory filters, no fabrication
O
Output
Comparison → Table, Time → Area Chart
E
Examples
2-3 query → response pairs anchoring behavior
Hands-On Lab — Build an Agent (20 min)
  1. Navigate to Home → Create → AI Agent
  2. Select your indexed dataset and configure column subsetting
  3. Write Supplemental Instructions using R.T.C.C.O.E:
    Role: You are a Sales Analytics Expert for Rogers. Task: Answer questions with accurate visualizations. Provide insight + recommendation. Context: Fiscal Year = Jun 1 – May 31. Q1 = Jun–Aug, Q2 = Sep–Nov, Q3 = Dec–Feb, Q4 = Mar–May. "current quarter" = FY2026 Q2. KPI: ATV = Sum(Sales_Revenue) / Count Distinct(Order ID). Grouping: Order Priority "Low, Medium, Not Specified" → "Regular" "High, Critical" → "Important" Constraints: Default metric = Revenue (Sum). Default ranking = Top 5. Never fabricate data. Output: Comparisons → Table. Time trends → Area Chart. Always show applied filters. Examples: "Revenue this quarter" → Filter Fiscal Quarter = 'FY2026 Q2', metric = Sum(Sales).
  4. Write a Welcome Message with 3 sample questions
  5. Upload a Knowledge Document (PDF with business context)
  6. Test your agent:
    • "Compare Average Profit for Regular and Important Orders"
    • "Show Revenue for Current Quarter"
    • "What's our ATV by Customer Segment?"
    • "Best 5 Products by Profit"

AI Functions
20 + 5 Q&A

The most valuable insights are often buried in free-text columns — customer comments, reviews, support tickets, agent notes, survey feedback. OAC AI Functions let you analyze these directly in the Expression Editor, no code or ETL required.

The Three AI Functions

AI_FILTER() — Find the Rows That Matter

Filters rows using semantic understanding, not keyword matching.

AI_FILTER("Customer Comments", 'complaints about delivery delays or shipping issues')

AI_GENERATE() — Classify Each Row

Generates a per-row value by applying an AI prompt to text.

AI_GENERATE('Classify as Positive, Negative, or Neutral', "Feedback")

AI_AGG() — Summarize the Broader Pattern

Aggregates across many rows into a consolidated summary.

AI_AGG('Summarize the top 3 recurring themes', "Feedback")
Hands-On Walkthrough
  1. Open a workbook with text-heavy data (customer comments, reviews, notes)
  2. Navigate to My Calculations → Add Calculation
  3. Use AI_GENERATE to classify each row
  4. Create a visualization using the classification column (e.g., pie chart of sentiment)
  5. Use AI_FILTER to drill into specific themes
  6. Apply AI_AGG to summarize patterns across groups
  7. Combine with standard OAC filters to slice by region, product, or time

OAC MCP Server
20 + 5 Q&A

The Model Context Protocol (MCP) Server connects OAC to external AI tools securely — extending governed analytics into Claude Desktop, Cline, GitHub Copilot, n8n, and LangChain.

Discover → Describe → Execute

ToolWhat It DoesExample
DiscoverLists available datasets and subject areas"What datasets are available?"
DescribeReturns tables, columns, measures, hierarchies"Describe the Sales dataset"
ExecuteRuns Logical SQL with full governance (RLS enforced)"Show revenue by region for 2025"
Setup Walkthrough (5 Steps)
  1. Generate Access Tokens: User Profile → Access Tokens → Download (tokens.json)
  2. Download MCP Connect: User Profile → MCP Connect → Download → Extract
  3. Install Node.js: v18+ from nodejs.org
  4. Configure Claude Desktop: Edit claude_desktop_config.json with OAC MCP server config
    { "mcpServers": { "oac-mcp-connect": { "command": "node", "args": ["/path/to/oac-mcp-connect.js", "https://<OAC-INSTANCE>.analytics.ocp.oraclecloud.com"], "description": "OAC MCP Server" } } }
  5. Verify: Open Claude Desktop → Check 3 tools connected (Discover, Describe, Execute)
Security: OAuth 2.0 authentication, role-based access, row-level security enforced during execution. Data governance maintained at the OAC layer.
